# <a name="host-tools-for-frontrow-events"></a>FrontRow 事件的主機工具

本文是以您在活動中啟用 FrontRow 的程式為基礎。 若要為您的活動啟用 FrontRow，請參閱這篇文章： [使用 FrontRow 功能調整您的使用者](../faqs/scaling-audiences.md)。

如果您想要瞭解適用于非 FrontRow 事件的主機工具，請參閱： [主機工具總覽](host-tools-overview.md)

現在您已將事件設定為 FrontRow，讓我們逐步瞭解如何使用 FrontRow 啟用的所有絕佳功能！

當您最初建立事件時，您已設定主機「角色」來處理該事件。 您也可以將主機角色提供給任何其他您想要用來協助執行事件的人，他們將擁有與您相同的裝載能力，您 (在此看到更多角色： [事件、世界、群組和 AltspaceVR) 中的角色](../getting-started/roles.md) 。 主機角色可讓您存取特別建立的特殊功能和功能，讓您輕鬆地裝載即時虛擬事件。

以下是在 AltspaceVR 中裝載 FrontRow 事件的這些功能、能力和其他實用提示的詳細總覽。

## <a name="host-tools-overview"></a>主機工具總覽

主機工具會出現在您現場的右下方。 您可以在建立事件之後立即使用它，如此一來，您就可以隨時輸入事件 (事件啟動時間不會套用到事件) 的主機或仲裁者，並在 showtime 前熟悉這些工具。 主機工具將可讓您快速點擊存取，例如，執行 FrontRow 事件所需的資訊，例如，執行事件時所需的 () 、對觀眾進行靜音和訊息處理、切換您的階段封鎖程式、自行 megaphoning、讓您的來賓能夠「舉手」以提出問題或提供意見反應、能夠從 FrontRow 房間移至房間等。

![使用醒目提示的可用動作開啟主機工具功能表](images/host-front-row-img-01.png)

*如果您看到 [進入無線操作] 按鈕，您可以將事件設定為 [FrontRow]。*

### <a name="toggle-mute-all"></a>切換 Mute-All

藉由靜音您的出席者，您可以限制干擾音效，並讓 FrontRow 活動中的東西保持美觀和安靜。 只要主機允許，或當主機離開空間時，靜音函式就會持續。 當出席者進入活動或嘗試取消靜音時，出席者將會收到畫面上的訊息，讓他們知道主機已經過靜音。 當主機 unmutes 時，出席者會收到一則訊息，告訴他們它們也 unmuted。

### <a name="message-all"></a>全部訊息

主機可以傳送訊息給其 FrontRow 事件中所有出席者。 使用我們的任何簡短附注，或撰寫一個原始的附注。 一旦傳送之後，每個人都會收到訊息作為快顯視窗。

### <a name="amplify-my-voice"></a>增強我的語音

主機可以增強其麥克風，以確保您的 Front Row 活動中的每個人都能聽到您所說的內容，不論距離為何。

### <a name="go-on-air"></a>進入無線

具有主機角色的任何人都可以看到此按鈕。 按一下 [移至] 會將您的頭像「鏡像」到所有 FrontRow 房間。 一旦您「鏡像」您所說的所有專案，就會在 FrontRow 事件的所有實例中看到。

在 FrontRow 事件的整個過程中，都是非常重要的。 您必須在無線上進行主機工具，才能在所有 FrontRow 房間內正常運作，否則只會在您所在的房間內運作。

> [!NOTE]
> 如果您需要重新加入您的活動，請移至 Campfire，並將您的網際網路連線中斷、從房間移至房間，然後當您回來時，您將不會在無線上進行。 當您返回之後，請確定您再次按一下 [移至無線]。

### <a name="toggle-stage-blocker"></a>切換階段封鎖程式

決定出席者可以待命的時機和位置。 選擇怪物焦點、輪流呈現，或邀請群組 dance 合作物件的階段上的所有人。

> [!IMPORTANT]
> 並非所有領域都包含階段封鎖程式，因此請明智地挑選您的事件範本。 建立您的官方活動之前，請先熟悉測試事件的事件範本。

## <a name="guest-panel"></a>來賓面板

可讓您追蹤事件「房間」中的每個人。 在 FrontRow 事件中，有多個房間，來賓面板只會顯示您目前所在房間內的來賓。 您可以按一下 [全部/開啟麥克風/主機/朋友/無線] 來篩選顯示的內容。 這是一份互動式清單，可讓您在您的房間內有多頁的明細。

嘗試尋找有不良音訊的人？ 開啟麥克風以查看誰是 unmuted。 想要確定您所需的每個人都已在 FrontRow 事件中進行鏡像嗎？ 選取 [無線] 以查看目前為無線和鏡像的所有人。 

### <a name="moderation-tools"></a>審核工具

作為事件主機，您必須負責管理他人的體驗。 若要有效率地進行仲裁，您和仲裁者需要健全的仲裁工具。 身為 FrontRow 事件的主機，您可以將干擾性的個體靜音、啟動事件的規則， (暫時或在事件) ，甚至是在事件期間將仲裁者/主機角色指派給其他人，以取得管理物件的協助。

您可以直接從 [來賓] 面板中按一下喇叭圖示旁邊的喇叭圖示來將來賓靜音，請選取位於來賓面板右邊的漢堡功能表：

![在 [來賓] 面板中選取單一來賓專案](images/host-front-row-img-03.png)

*按一下漢堡功能表以開啟所有審核工具選項。*

![已反白顯示審核工具選項所選取的單一來賓專案](images/host-front-row-img-04.png)

現在您可以使用 FrontRow 裝載魔術，並且能夠在您的出席者上使用這些功能：
    * **訊息** -一般而言，您必須是朋友才能傳送給其他使用者，但作為主機，您可以在您的活動中傳送/接收訊息給任何人。
    * 為事件中的每個人將 **其靜音。**
    * **擴音器** -讓他們能夠增強語音。
    * **在無線上** -您可以「鏡像」這項來賓，如此一來，FrontRow 事件的所有房間中的所有人都能看到自己的圖片和聲音。
    * **暫存存取** -讓他們能夠完成任何階段封鎖程式，並進入階段區域。
    * **Make host** -讓它們成為您 FrontRow 事件中的主機角色。 立即新增 Co-Host，確定它們已進入空中，並可協助您執行專案，或讓他們移至其他房間來檢查事物。 每個人都是在飛機上，並且顯示在次要房間內嗎？ 您的簡報是否會在所有作用中的房間之間同步？ 當他們在其他房間內時，讓他們進入空氣中，您就能夠從房間到房間的地點討論頭像！
    * **讓仲裁** 者在您的 Front Row 活動中提供仲裁者角色，以協助您進行控制。 仲裁者將可存取有限的主機工具，因此可以從空間移至空間來檢查事物，但不需要進行無線連線。 
    * **啟動** 規則-從您的事件暫時或在您的事件期間中斷

## <a name="participation-panel"></a>參與面板
 
您可以使用此面板來協助您與您的活動出席者互動。 您可以有 Q&A、表單要針對開啟的 Mic 執行的佇列，或確定您正在交談的來賓也會進行鏡像，讓每個人都能聽到您彼此在所有 FrontRow 房間內的對話：

![已選取參與索引標籤的 [開啟主機面板]](images/host-front-row-img-05.png)

參與面板的設計目的是協助您管理使用者的互動和參與。 您可以在這裡切換 [允許手動引發] 選項，以解除鎖定物件參與。 **在 [FrontRow 事件] 中，您必須先在空氣中啟用此功能，否則這個按鈕會呈現灰色。**

當啟用「允許舉手」功能時，FrontRow 事件中的每個人都會出現「舉手」按鈕，其位於現場的右下角區段中：

![在參與面板中醒目提示的舉手函數](images/host-front-row-img-06.png)

*按一下 [舉手] 會將來賓放在您所控制的佇列中。*

當來賓按一下這個按鈕時，它們會顯示在您的參與面板中：

![使用來賓開啟並醒目提示佇列中的參與面板](images/host-front-row-img-07.png)

現在您可以按一下 [無線 + 擴音器] 按鈕來呼叫它們，這會在啟用之後開啟藍色按鈕。 此動作會將兩則訊息傳送至手中，以確認他們想要取消靜音和「主機已呼叫您，並使您變得更大」。 一旦接受之後，按鈕就會變成藍色，這會讓它們進入無線「鏡像」，並增強我的聲音「Megaphoned」以查看並聆聽！

當您完成與其互動之後，請先按一下藍色按鈕將它們取出，然後按鈕將不再是藍色，然後按一下 X 以關閉清單中的按鈕。 現在您已備妥幸運 contestant 第二個，依此類推： )

## <a name="room-panel"></a>房間面板

迅速查看您在 FrontRow 事件期間管理房間所需的資訊/按鈕：

![從主機面板開啟的 [會議室] 面板索引標籤](images/host-front-row-img-08.png)

*請注意， **(您在這裡)**，也就是您目前所在的房間。*

會議室具有重設空間、刪除空間、重新發佈使用者和傳送至該處的按鈕：

![在房間面板中醒目提示 [重設空間] 按鈕](images/host-front-row-img-09.png)

會在您目前所在的房間內進行「重設空間」。 您無法一次重設所有房間的空間，您必須移動到每個房間以進行重設。

![[空間] 面板中醒目提示的 [刪除空間] 按鈕](images/host-front-row-img-10.png)

永久刪除您目前所在的房間。 已刪除房間中的每個人都會自動重新加入活動，並加入不同的房間。

![在 [聊天室] 面板中醒目提示 [重新發佈使用者] 按鈕](images/host-front-row-img-11.png)

有時，您的 FrontRow 事件一段時間之後，前幾個房間幾乎都是容量，但最後一個房間或兩個房間可能只有少數來賓的每個房間。 按一下此按鈕可讓該房間內的來賓選擇保持在目前的房間內，或前往更擴展的房間。

> [!NOTE]
> 您不需要移動到您想要重新發佈的房間，只要在會議室面板中按一下任何房間的 [轉散發使用者] 按鈕即可。 請不要對相同的房間進行太多次的動作，可能會想要使用一些人員，而不想要繼續取得要求移動房間的訊息。

![在 [聊天室] 面板中醒目提示 [傳送] 按鈕](images/host-front-row-img-12.png)

這是您從一個 Front 資料列房間移動到另一個房間的方式。 仲裁者會想要使用這種從房間到房間的地方，來檢查 crowds。 在 rehearsals 期間，您可以在主要房間內擁有一部主機，並讓其餘的小組移至其他房間，以查看所有專案的外觀。 所有專案都在其他房間內同步，而且看起來像是在主要房間內嗎？ 如果協助測試的每個人都有主機角色，也可以移至其他房間，進入其他房間，您將能夠看到並聆聽所有人，讓您能夠以這種方式在會議室之間進行通訊。
